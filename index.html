Fundamentos: ¿Por qué aprender esto?

Imagina que los datos son como un montón de piezas de LEGO. Aprender a analizarlos es como aprender a construir cualquier cosa que imagines con esas piezas. Es una habilidad súper demandada que te abre las puertas a trabajos increíbles y bien pagados.

Definición: Python es un lenguaje de programación. Piensa en él como el idioma que usaremos para darle instrucciones a la computadora. Es famoso por ser muy claro y fácil de leer, ¡casi como leer inglés!
¿Por qué aprenderlo?: Es el lenguaje #1 para Ciencia de Datos, Machine Learning e Inteligencia Artificial. Su comunidad ha creado miles de "librerías" (cajas de herramientas pre-hechas) que nos ahorran muchísimo trabajo.
Ejemplo: total_empleados = df['nombre'].count(). Esto es como decirle a Python: "Oye, del archivo de datos, cuenta cuántos nombres hay y dime el total".

Definición: Streamlit es una librería de Python que hace que crear aplicaciones web sea increíblemente fácil y rápido. Es como tener una varita mágica que convierte tu código de análisis en una página web interactiva.
¿Por qué aprenderlo?: Te permite compartir tus análisis y gráficos con cualquier persona (tu jefe, tus amigos, un cliente) sin que ellos necesiten saber programar. ¡Simplemente les pasas un link!
Ejemplo: st.slider("Rango de Edad", 18, 65). Con esta simple línea, Streamlit crea un control deslizante en la web para que el usuario pueda filtrar por edad.

Definición: Pandas es la librería para trabajar con datos tabulares (piensa en tablas de Excel o de bases de datos). Su herramienta principal es el DataFrame, que es como una hoja de cálculo súper poderosa que controlas con código.
¿Por qué aprenderlo?: Es la navaja suiza del analista de datos. La usarás para TODO: leer archivos, limpiar datos con errores, filtrar, agrupar, y hacer cálculos.
Ejemplo: df_filtrado = df[df['departamento'] == 'Ventas']. Aquí le decimos a Pandas: "Del DataFrame completo, dame solo las filas donde el departamento sea 'Ventas'".

Definición: Plotly es una librería para crear gráficos interactivos y hermosos. A diferencia de una imagen estática, los gráficos de Plotly te permiten pasar el mouse para ver detalles, hacer zoom, y explorar la información.
¿Por qué aprenderlo?: Un buen gráfico vale más que mil palabras. Plotly te ayuda a contar la historia que esconden tus datos de una forma visual y atractiva, permitiendo que otros descubran patrones por sí mismos.
Ejemplo: fig = px.histogram(df, x="departamento"). Con esto, Plotly crea un gráfico de barras que cuenta cuántos empleados hay en cada departamento.

Bloque 3: Carga y Limpieza de Datos (La Función Mágica)

🧠 En este Bloque 3, presentamos la función que actúa como el corazón del dashboard. Su misión es clara: leer el archivo Excel, limpiar los datos y dejarlos listos para el análisis. Este proceso garantiza que la información esté libre de errores, con nombres de columnas estandarizados y formatos coherentes para su uso en visualizaciones.

⚡ Para optimizar el rendimiento, usamos el decorador @st.cache_data, una herramienta inteligente de Streamlit que detecta si el archivo ha cambiado. Si no hay modificaciones, evita volver a cargar y procesar los datos, lo que convierte a la app en una experiencia ágil, eficiente y ultrarrápida.

🔍 Además, la función realiza tareas clave como convertir fechas, calcular edades y antigüedad laboral, y transformar valores monetarios en formato numérico. Todo esto se hace de forma robusta y controlada, asegurando que el análisis posterior se base en datos limpios, coherentes y listos para visualizar.

📁 Este archivo contiene el set de datos original del área de Recursos Humanos, incluyendo información sobre sueldos, bonificaciones, fechas clave y datos personales anonimizados. Es la base para todas las visualizaciones y cálculos del dashboard. ¡Descárgalo para explorar, practicar o personalizar tu propio análisis! 🔍📊 


# Usamos un decorador para que la función se ejecute solo una vez si los datos no cambian
@st.cache_data
def cargar_datos(ruta_archivo: str) -> pd.DataFrame:
    try:
        df = pd.read_excel(ruta_archivo)
    except FileNotFoundError:
        st.error(f"Error: No se encontró el archivo '{ruta_archivo}'.")
        return pd.DataFrame()
        
    df.columns = df.columns.str.strip().str.lower().str.replace(' ', '_')
    df = df.rename(columns={'bonificación': 'bonificacion', 'edad_en_años': 'edad_calculada', 'nombre_completo': 'nombre_completo'})
    
    for col in ["sueldo", "bonificacion", "total_ingresos"]:
        if col in df.columns and df[col].dtype == 'object':
             df[col] = df[col].replace(r'[\$,]', '', regex=True).astype(float)
             
    for col in ['fecha_nacimiento', 'fecha_ingreso', 'fecha_salida']:
        if col in df.columns:
            df[col] = pd.to_datetime(df[col], errors='coerce')
            
    hoy = pd.Timestamp.now()
    if 'fecha_nacimiento' in df.columns: 
        df['edad_en_años'] = (hoy - df['fecha_nacimiento']).dt.days / 365.25
    else: 
        df['edad_en_años'] = np.nan
        
    if 'fecha_ingreso' in df.columns: 
        df["antiguedad_num"] = ((hoy - df["fecha_ingreso"]).dt.days / 365.25).fillna(0)
    else: 
        df["antiguedad_num"] = 0
        
    return df

df_original = cargar_datos("RRHH.xlsx")

if df_original.empty: 
    st.stop()

📊 Bloque 5.01: KPIs y Filtrado de Datos

Esta sección del dashboard tiene como objetivo central el filtrado de datos. A través de los filtros configurados (departamentos, género, cargos, rango de edad y antigüedad), logramos mostrar solo la información relevante para el análisis. Esto permite personalizar la vista y concentrarse únicamente en el subconjunto de empleados que interesa al usuario.

Una vez aplicado el filtrado, se calculan los KPIs más importantes: número total de empleados, activos, inactivos, sueldo total, edad promedio, cantidad de departamentos y la tasa de rotación. Estos indicadores, además de ser intuitivos, ofrecen una visión rápida del estado actual de la organización. Se presentan en métricas visuales que permiten la comparación inmediata entre diferentes dimensiones de los datos.

El resultado es un bloque fundamental, ya que combina filtrado dinámico con métricas clave, convirtiéndose en la base para los análisis posteriores del dashboard. Gracias a estos cálculos iniciales, los usuarios pueden tomar decisiones mejor fundamentadas sobre el personal de la empresa. 


# =====================
# 📊 BLOQUE 5.01: KPIs y Filtrado de Datos

# --- Filtrado del DataFrame con .query() ---
query_parts = [
    "departamento in @departamentos",
    "genero in @generos",
    "cargo in @cargos",
    "edad_en_años >= @rango_edad[0] and edad_en_años <= @rango_edad[1]",
    "antiguedad_num >= @rango_antiguedad[0] and antiguedad_num <= @rango_antiguedad[1]"
]
df_filtrado = df_original.query(" and ".join(query_parts))

# Verificar si hay datos después del filtrado
if df_filtrado.empty:
    st.warning("⚠️ No hay datos que coincidan con los filtros seleccionados.")
else:
    # --- KPIs ---
    # Se ajusta el estilo de las métricas (fuentes y tamaño)
    st.markdown("""""", unsafe_allow_html=True)
    
    # Se dividen los KPIs en columnas para mostrarlos en línea
    kpi_cols = st.columns(7)
    
    # Cálculo de KPIs principales
    total_empleados, activos = len(df_filtrado), df_filtrado["estado"].str.lower().eq("activo").sum()
    inactivos = total_empleados - activos
    tasa_rotacion = (inactivos / total_empleados * 100) if total_empleados > 0 else 0
    
    # Mostrar métricas

# Función: cálculo de rotación anual
def calcular_rotacion_anual(df: pd.DataFrame) -> pd.DataFrame:
    df_salidas = df.dropna(subset=['fecha_salida']).copy()
    if df_salidas.empty: return pd.DataFrame()
    df_salidas['año_salida'] = df_salidas['fecha_salida'].dt.year
    separaciones_anuales = df_salidas.groupby('año_salida').size()

    def calc_avg_employees(year):
        # Se calcula el promedio de empleados al inicio y al final del año
        start_of_year = df[(df['fecha_ingreso'].dt.year < year) & ((df['fecha_salida'].dt.year >= year) | (df['fecha_salida'].isnull()))].shape[0]
        end_of_year = df[df['fecha_ingreso'].dt.year <= year].shape[0] - df[df['fecha_salida'].dt.year <= year].shape[0]
        return (start_of_year + end_of_year) / 2 if (start_of_year + end_of_year) > 0 else 1

    promedio_empleados_anual = {year: calc_avg_employees(year) for year in separaciones_anuales.index}
    rotacion_df = pd.DataFrame({'separaciones': separaciones_anuales, 'promedio_empleados': pd.Series(promedio_empleados_anual)})
    rotacion_df['tasa_rotacion'] = (rotacion_df['separaciones'] / rotacion_df['promedio_empleados']) * 100
    return rotacion_df.reset_index().rename(columns={'index': 'Año'})

# Función: cálculo de rotación por departamento
def calcular_rotacion_departamento(df: pd.DataFrame) -> pd.DataFrame:
    rotacion = df.groupby('departamento')['estado'].apply(lambda x: (x.str.lower() == 'inactivo').sum() / len(x) * 100 if len(x) > 0 else 0)
    return rotacion.reset_index(name='Tasa de Rotación (%)').sort_values('Tasa de Rotación (%)', ascending=False)

        st.plotly_chart(px.bar(rotacion_depto_df, x='departamento', y='Tasa de Rotación (%)', title='Tasa de Rotación por Departamento (%)'), use_container_width=True)

# Función para calcular la antigüedad de un empleado
def calcular_antiguedad_texto(row):
    final_date = row["fecha_salida"] if pd.notna(row["fecha_salida"]) else pd.Timestamp.now()
    if pd.notna(row["fecha_ingreso"]):
        delta = final_date - row["fecha_ingreso"]
        años, dias_restantes = divmod(delta.days, 365)
        meses, _ = divmod(dias_restantes, 30)
        return f"{años} años, {meses} meses"
    return "N/A"

# Crear una copia de los datos y añadir columnas calculadas
